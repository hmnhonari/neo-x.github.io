Ubuntu 12.04 on ASUS n56v

Feb 7, 2013 • Glen B

This blog is a collection of solutions to issues or not preferred settings in Ubuntu when installing on ASUS n56v

Press f2 to get into bios

<ol><li>After installing Ubuntu fix Grub to boot Windows 8 again.</li><ol>So installing Ubuntu 12.04 along with a EFI partition with GRUB will result in a boot error for Windows8 (from /dev/sda[some number]), some thing like: error: unknown command ‘drivemap’ error: invalid EFI file path. While there are many threads online discussed about this issue. Here is the most elegant solution I found: using the Boot-Repair tool. Here is how: </ol><ol> 1. After Ubuntu is installed (make sure you can access the internet), open the terminal:

sudo add-apt-repository ppa:yannubuntu/boot-repair
sudo apt-get update </ol><ol> 2. Press Enter. </ol><ol> 3. Then type:</ol><ol> </ol><ol> sudo apt-get install -y boot-repair</ol><ol> boot-repair </ol><ol> 4. Press Enter, let this software figure everything out for you.</ol><ol>It will automatically generate the GRUB files such that your Windoes8 can boot again.</ol><ol> </ol><ol>Best to choose the Recommended option to fix your boot issues in boot-repair </ol><ol>

For whatever reason the computer seem to freeze every so often. Looks to occur whenever I go to save something.

Apparently it can be easily remedied by enabling the long term service hardware stack like so:

sudo apt-get install linux-generic-lts-quantal xserver-xorg-lts-quantal
After this you should have kernel version >= 3.5.0 which is the point of enabling the hardware support.
</li><ol></ol><li>Fix auto brightness
<div class="post-text">You could try adding a line to /etc/rc.local that will set the desired brightness level. To edit the file, run
<pre>gksudo gedit /etc/rc.local
</pre>and add the following
<pre>echo X > /sys/class/backlight/intel_backlight/brightness
</pre>so that the end result looks like this
<pre>#!/bin/sh -e
#
# rc.local
#
# This script is executed at the end of each multiuser runlevel.
# Make sure that the script will "exit 0" on success or any other
# value on error.
#
# In order to enable or disable this script just change the execution
# bits.
#
# By default this script does nothing. </pre><pre> </pre><pre># Max vlaue seems to be 4882 </pre><pre> echo X > /sys/class/backlight/intel_backlight/brightness

exit 0
</pre>Substitute the X by the desired brightness level.</div>
